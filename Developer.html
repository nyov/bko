How to replicate the BKO setup on your machine?

  1. Get the BKO.git and gpxe repositories. You can use the script setup.sh for this work.
  2. Edit the BKO/configure to suit your needs. It can be done with any text editor 
    vi BKO/configure
    Make sure that you provide proper values for atleast GPXE_GIT_LOCATION, BOOT_URL and ISO_LOCATION_LOCAL Other entries are optional, you can leave them untouched. Please refer bellow for more details on the meaning of the configuration variables.
  3. Follow the following instructions 
    cd BKO/install_help
./deploy.sh
./download_ISO.sh

Details about configuration parameters

  1. GPXE_GIT_LOCATION: path to the place where gpxe git is placed. If you have used the setup.sh script, then this path should be ../gpxe. the gpxe folder can be moved anywhere as long as the path is given properly.
  2. BOOT_URL: gpxe will contact this URL for initial boot. It should point to the BKO folder which is created by setup.sh script. and it must be HTTP URL and not linux path.
  3. ISO_LOCATION_LOCAL : The HTTP path where all ISO's will be downloaded. the script download_ISO.sh will download them in folder named ISO in the same location where setup.sh was executed. For example, if you have executed setup.sh in topmost directory (ie var/www/) then BOOT_URL=http://your-ip-address/BKO/ and ISO_LOCATION_LOCAL=http://your-ip-address/ISO/

Details about installation help scripts

  1. setup.sh : It is the first script that user will need. It clones the git repository which can be used for local deployment.
  2. deploy_gpxe.sh : It configures and compiles the gpxe at location specified by GPXE_GIT_LOCATION for contacting the BOOT_URL for fetching pxelinux.0
  3. configure_BKO.sh : BKO menus are presented with help from configuration files. These configuration files .conf also includes so deployment specific information like ISO_LOCATION_LOCAL. This script genenates all these configuration files with correct information.
  4. deploy_BKO.sh : This script is responsible for generation of all the initramfs files, needed by all supported distributions. This script do require root privileges for initramfs generation. This script uses sudo whenever it need root privileges, so user may have to enter the password more than once. User can also execute entire script as root. In case you do not have such privilages, you can use download_initramfs_images_http.sh which will download these initramfs images from server for you.
  5. download_initramfs_images_http.sh : Downloads the modified initramfs images which supports httpfs booting. These initramfs images are downloaded from REMOTE_INITRAMFS_DOWNLOAD_LOCATION. The default value for this variable will be correct one, so users dont need to temper with it.
  6. download_ISO.sh : BKO depends on ISO images of various distributions for its work. This script download those ISO's on your behalf.
