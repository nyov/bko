boot.kernel.org(BKO):Booting your machie over HTTP

Our objective is to let users boot their machines with bare minimal requirements on their side, and we will handle most of the problem that you may face in booting. All that user needs is Internet connectivity and a small program (gpxe) to boot the machine. This gpxe program provides network booting facility. You can download these program from our site (available at download section).

We will also give users choices between various destributions of linux to choose from. User also may choose from various rescue tools which can help in data recovary and in other situations. The list of existing choices is available here, and we assure you that this list will grow over time :-)

How to use it?

  1. Download the gpxe image
  2. copy it in floppy, USB or in CDROM (exactly how?)
  3. Make sure that you know what kind of network you have (Which entries will be needed)
  4. Boot from the media in which you had copied the gpxe Image
  5. In case of Static networking, you need to provide information about your connection, but if you are having DHCP, it should work out of the box.
  6. You will be presented with menu showing available options to boot from, select one of them.
  7. Your system should boot in the sytem you select
  8. Once you are done with it, reboot the system and remvoe the media, you should get back your old system

Downloads

gpxe image for floppy
Once downloaded, you need to copy this image to floppy with command 

cat gpxe.dsk > /dev/fda
If you just want to test it, or get feel of it, then you can use it with qemu with following command. 
qemu -fda gpxe.dsk
